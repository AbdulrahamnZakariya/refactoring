02. عملية إعادة التصميم (Refactoring)

المدة: 11.56 دقيقة

إعادة التصميم هي عملية تحسين الكود البرمجي دون تغيير وظيفته. تهدف إلى تحسين بنية الكود وجعله أكثر قابلية للقراءة والصيانة. تشمل العملية إعادة تنظيم الكود، إزالة التكرار، وتبسيط الهيكل العام.

أهمية إعادة التصميم:

  • زيادة وضوح الكود وسهولة فهمه.
  • تقليل الأخطاء البرمجية وجعل الكود أكثر استقرارًا.
  • تعزيز إمكانية إضافة ميزات جديدة دون التعرض لمشاكل معقدة.

أفضل الممارسات:

  • القيام بإعادة التصميم بشكل تدريجي ومتكرر.
  • إجراء اختبارات بعد كل تعديل لضمان عدم التأثير على وظائف النظام.

03. دور عملية إعادة التصميم في منهجية Agile

المدة: 8.30 دقيقة

تعتبر إعادة التصميم جزءًا أساسيًا في منهجية Agile، حيث يتم تحسين الكود بشكل مستمر كجزء من دورات التطوير القصيرة. تُسهل هذه العملية التعامل مع التغييرات المستمرة في المتطلبات.

فوائد إعادة التصميم في Agile:

  • تمكين الفريق من التعامل مع تغييرات المتطلبات بسرعة.
  • ضمان أن يظل الكود مرنًا وقابلًا للتطوير.
  • تعزيز التعاون بين أعضاء الفريق من خلال كود منظم.

01. التعامل مع المتغيرات: استخراج المتغير (Extract Variable) وتسطير المتغير (Inline Variable)

المدة: 9.16

تعد عملية استخراج المتغير (Extract Variable) مفيدة عندما يتم تكرار تعبير معين في أكثر من مكان. من خلال استخراج المتغير، يمكن تحسين وضوح الكود. بينما عملية تسطير المتغير (Inline Variable) تعمل على إزالة المتغيرات غير الضرورية.

مثال على استخراج المتغير:

                // قبل استخلاص المتغير
                let total = price * (1 + taxRate);
                
                // بعد استخلاص المتغير
                let taxAmount = price * taxRate;
                let total = price + taxAmount;

02. التعامل مع المتغيرات: تغليف المتغير (Encapsulate Variable) وإعادة تسمية المتغير (Rename Variable)

المدة: 8.25

التغليف (Encapsulate) يمكن أن يساعد في حماية المتغيرات من التعديلات المباشرة، مما يسهل إدارة القيم المحدثة. أما إعادة التسمية، فتهدف إلى تحسين وضوح الكود عن طريق استخدام أسماء واضحة.

مثال على تغليف المتغير:

                // قبل التغليف
                let discount = 0.2;
                if (discount > 0.3) { 
                    // some logic 
                }

                // بعد التغليف
                class Product {
                    private _discount = 0.2;
                    
                    get discount() {
                        return this._discount;
                    }

                    set discount(value) {
                        if (value > 0.3) {
                            this._discount = value;
                        }
                    }
                }

03. التعامل مع الدوال - الجزء الأول

المدة: 14.11

يتعامل هذا الجزء مع كيفية تحسين الدوال، مثل تقليص حجم الدوال الطويلة وجعلها أكثر وضوحًا. مثال على كيفية إعادة هيكلة دالة طويلة.

مثال على تقليص دالة طويلة:

                // قبل التحسين
                function calculateOrder(price, quantity, discount) {
                    let total = price * quantity;
                    total = total - (total * discount);
                    if (total > 1000) {
                        total = total - 100;
                    }
                    return total;
                }

                // بعد التحسين
                function calculateTotal(price, quantity) {
                    return price * quantity;
                }

                function applyDiscount(total, discount) {
                    return total - (total * discount);
                }

                function applyShippingFee(total) {
                    if (total > 1000) {
                        return total - 100;
                    }
                    return total;
                }

                function calculateOrder(price, quantity, discount) {
                    let total = calculateTotal(price, quantity);
                    total = applyDiscount(total, discount);
                    return applyShippingFee(total);
                }

04. التعامل مع الدوال: تغيير عبارة التصريح وإضافة كائن المعامل

المدة: 15.55

يمكن تغيير تصريح الدالة لإضافة كائن المعامل، مما يسهل إضافة خصائص إضافية في المستقبل دون التأثير على التوقيع العام.

مثال على إضافة كائن المعامل:

                // قبل التعديل
                function createOrder(price, quantity, discount) {
                    return price * quantity - discount;
                }

                // بعد التعديل
                function createOrder({ price, quantity, discount = 0 }) {
                    return price * quantity - discount;
                }

05. التعامل مع الدوال: دمج الدوال في صنف

المدة: 6.50

تتمثل هذه التقنية في دمج الدوال التي تتعامل مع نفس الكائن في صنف واحد، مما يسهل التعامل مع البيانات.

مثال على دمج الدوال في صنف:

                // قبل الدمج
                function calculateTotal(price, quantity) {
                    return price * quantity;
                }

                function applyDiscount(total, discount) {
                    return total - (total * discount);
                }

                function createOrder(price, quantity, discount) {
                    let total = calculateTotal(price, quantity);
                    return applyDiscount(total, discount);
                }

                // بعد الدمج
                class Order {
                    constructor(price, quantity) {
                        this.price = price;
                        this.quantity = quantity;
                    }

                    calculateTotal() {
                        return this.price * this.quantity;
                    }

                    applyDiscount(discount) {
                        return this.calculateTotal() - (this.calculateTotal() * discount);
                    }
                }

                let order = new Order(100, 2);
                console.log(order.applyDiscount(0.1));

06. التعامل مع الحلقات التكرارية

المدة: 15.17

مثال:

                // قبل التحسين
                for (let i = 0; i < items.length; i++) {
                    console.log(items[i]);
                }

                // بعد التحسين (باستخدام دالة map)
                items.map(item => console.log(item));

07. التعامل مع العبارات الشرطية

المدة: 13.57

تحسين العبارات الشرطية عن طريق تجنب التكرار واستخدام الكود بطريقة أفضل.

مثال:

                // قبل التحسين
                if (age < 18) {
                    console.log('مراهق');
                } else if (age >= 18 && age <= 65) {
                    console.log('بالغ');
                } else {
                    console.log('مسن');
                }

                // بعد التحسين
                switch (true) {
                    case (age < 18):
                        console.log('مراهق');
                        break;
                    case (age <= 65):
                        console.log('بالغ');
                        break;
                    default:
                        console.log('مسن');
                }

02. البرمجة الموجهة بالاختبارات

المدة: 22.31

البرمجة الموجهة بالاختبارات (Test-Driven Development - TDD) هي منهجية تطوير البرمجيات التي تعتمد على كتابة الاختبارات قبل كتابة الكود. يهدف هذا النهج إلى ضمان الجودة وتحسين صيانة الكود.

مثال عملي:

                    // كتابة الاختبار أولاً
                    function add(a, b) {
                        return a + b;
                    }
    
                    // اختبار دالة add
                    describe('add function', () => {
                        it('should add two numbers correctly', () => {
                            assert.equal(add(2, 3), 5);
                        });
                    });

03. البرمجة الموجهة بالاختبارات في جافاسكربت

المدة: 10.41

في جافاسكربت، يتم استخدام إطار العمل مثل Mocha أو Jasmine لتنفيذ البرمجة الموجهة بالاختبارات. يتناول هذا القسم كيفية تطبيق هذا المنهج باستخدام الأدوات المتاحة في جافاسكربت.

مثال عملي باستخدام Mocha:

                    const assert = require('assert');
    
                    // دالة للتنفيذ
                    function subtract(a, b) {
                        return a - b;
                    }
    
                    // كتابة اختبار
                    describe('subtract function', () => {
                        it('should subtract two numbers correctly', () => {
                            assert.equal(subtract(5, 3), 2);
                        });
                    });
